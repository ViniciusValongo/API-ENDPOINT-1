A aplição foi desenvolvida em flask e ao criar as imagens dos microsserviços no docker o nginx não funciona quando o container de backup é criado, porém sem esse container a aplicação funciona normalmente e o container com peso 3 suporta mais solicitações que o container padrão peso 1. O container de backup só entrará em ação caso os 2 anteriores estejam indisponíveis. Caso nenhum esteja funcionando o sistema retornará status 502.
